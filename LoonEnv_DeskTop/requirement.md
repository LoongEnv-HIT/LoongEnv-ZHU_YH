你正在参与 LoongEnv 项目的开发。

【项目目标】
LoongEnv 是一个面向多类型工业机器人的 AI 辅助开发平台，目标是建立从机器人定义到实际运行与持续优化的统一工程闭环，提高开发效率、可复现性与系统稳定性。

支持对象包括：

* 关节型机械臂
* 移动机器人（AGV）
* 复合机器人（移动+操作）
* 双臂与移动双臂系统

---

【核心架构：4D + Core】

系统采用以下主线：

Define → Design → Deploy → Diagnose（闭环）

* Define：定义机器人结构、模型与任务
* Design：设计控制与算法，并完成仿真验证
* Deploy：将算法部署到真实系统进行运行
* Diagnose：监测运行状态，分析问题并生成改进依据

Core：贯穿所有阶段的数据与模型底座（state/action/log/模型）

---

【开发原则】

* 所有功能必须明确归属 4D 某一阶段
* Design 阶段必须完成验证后才能进入 Deploy
* Python 不进入实时控制环（实时控制由 C++ 承担）
* 所有运行数据必须记录为统一 log（用于 Diagnose）
* Diagnose 的结果必须能够回流到 Design 形成闭环优化
* AI 仅作为辅助工具，所有关键决策必须可审查、可追溯

---

【你的任务要求】

在进行任何开发、设计或分析时：

1. 明确当前工作属于哪个阶段（Define / Design / Deploy / Diagnose）
2. 明确输入与输出（数据/模型/接口）
3. 保持模块解耦，不跨层混用职责
4. 优先保证结构清晰与可扩展性，而不是功能复杂度
5. 所有输出应符合工程化标准，可被团队复用

这是一个“系统工程平台”，不是单点功能开发。
